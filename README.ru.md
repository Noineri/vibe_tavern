## Почему Vibe Tavern?

Vibe Tavern — это локальная платформа для AI-ролплея, написанная с нуля: настоящая база данных вместо плоских JSON-файлов, типизированный монорепозиторий вместо монолитного скрипта и интерфейс, который не лезет в глаза и не требует вики для понимания.

Ты используешь свои API-ключи. Данные остаются на твоей машине. Интерфейс остаётся на втором плане.

> **Статус:** 🛠️ Beta-2. Базовый ролплей стабилен и протестирован. Часть продвинутых вещей ещё в работе — см. [Roadmap](#roadmap).

> [!NOTE]
> Проект в активной разработке — жди новых фич, иногда ломающих изменений и периодических багов. Не забывай делать бэкап директории `data/` перед обновлением.

## Вкратце

Для тех, кто зашёл на 5 секунд:

- **Версии персонажа** — создавайте параллельные ветки одного персонажа, редактируйте их независимо и переключайтесь между ними в один клик.
- **12-гейтовый движок лорбуков** — активация по ключевым словам с возможностью трассировки, рекурсии, закрепления окон (sticky windows) и вытеснения по приоритету.
- **Визуальный холст промптов** — настраивайте порядок слоев промпта простым перетаскиванием, включая инжекты на определенную глубину контекста.
- **Встроенное сжатие контекста** — автоматическая саммаризация текста с визуальным отображением токенов, без необходимости ставить сторонние расширения.
- **Трассировщик промптов** — послойный разбор собранного промпта с количеством токенов и источником активации на каждый слой.
- **Один `.exe` + доступ с телефона по QR** — всё локально, твои ключи, твои данные, твоя машина.

Длинная версия — ниже.

## Фичи ⭐

### Чат

- **Ничего лишнего** — только ты и персонаж. Переключай чаты, персоны, пресеты промптов и модели из быстрого доступа, не отвлекаясь от общения.
- **Стриминг с рассуждениями** — блоки reasoning свёрнуты по умолчанию, чтобы не мешать. Но открой — и увидишь, что размышления выглядят стильно.
- **Свайпы вариантов ответа** — десктопная анимация слайдов + мобильная карусель из трёх панелей. На каждом свайпе — подсчёт токенов и лейбл модели.
- **Подсчёт токенов прямо в поле ввода** — следи за бюджетом контекста, не открывая настройки. Цветовая индикация + всплывающее окно, где видно, что сколько съедает (системный промпт, лорбук, история и т.д.).
- **Очередь генераций** — складывай генерации в стопку и иди по своим делам. Каждый вариант запоминает, какой моделью и пресетом он был сгенерирован.

### Редактор персонажа

- **Двойной режим редактирования** — переключайся между структурной формой и полноценным Markdown-редактором (CodeMirror 6) на одном персонаже в любой момент. Изменения синхронизируются в обе стороны.
- **Структурная фиксация** — Markdown-редактор жёстко держит четыре канонические секции (`# PERSONALITY`, `# SCENARIO`, `# EXAMPLES`, `# GREETINGS`). Случайно удалить их не выйдет — ни тебе, ни ИИ-соавтору.
- **Версии персонажа** — ветвь персонажа на параллельные редактируемые варианты (v1 «База», v2 «Агрессивный», v3 «Романтичный»). Переключение мгновенное, каждая версия — полный снимок.
- **Импорт карточек SillyTavern V2/V3** — PNG с встроенным JSON, массовый импорт из директории ST, экспорт без потерь.
- **Markdown-импорт через ИИ** — вставь хаотичный черновик (заметки, прозу, лог чата) — и ИИ-ассистент раскидает его по формализованным полям. Если модель что-то распознала криво — не беда: смени таргет промпта. Если всё совсем плохо, снизь температуру или поменяй модель.

### Лорбуки

- **12-шаговый движок активации** — ключевые слова с логикой AND/OR/NOT, вторичные ключи, кулдауны, липкие окна, задержка, вероятность, приоритетное вытеснение и рекурсивное сканирование (лор, активирующий лор).
- **Упрощённый режим по умолчанию** — чтобы быстро набросать запись, видно только ключи и текст. Полный редактор (полная копия такового из SillyTavern) разворачивается по кнопке — со всеми тонкими настройками.
- **Трейс активации** — видно точно, какие записи сработали и почему, прямо в промпт-трейсере. Никаких догадок.
- **Генерация ключей через ИИ** — боишься чистого листа? Попроси ИИ-ассистента сгенерировать первичные и/или вторичные ключи по тексту записи.
- **Привязка к скоупу** — лорбуки крепятся к персонажам, персонам или работают глобально. Связь «многие-ко-многим» через таблицу — один лорбук на несколько персонажей.
- **ST-совместимый импорт/экспорт** — полный паритет с форматом лорбуков SillyTavern.

### Конвейер промптов

- **Промпт-трейсер** — посмотри каждый слой собранного промпта: system, jailbreak, описание персонажа, инъекции лорбука, сводки, author's note, кастомные инъекции на глубину. На каждый слой — токены, глубина и источник активации.
- **Пресеты промптов** — полный контроль над system prompt, jailbreak, prefill, author's note (с настраиваемой глубиной), промптом сводки и tools-промптом. Импорт/экспорт ST-совместимых пресетов.
- **Продвинутое упорядочивание** (Канвас) — визуальный редактор позиций инъекций с перетаскиванием. Три зоны: до чата, внутри чата (на конкретной глубине), после чата. Визуальная позиция — источник истины.
- **Полный движок макросов ST** — `{{user}}`, `{{char}}`, `{{if}}`, `{{setvar}}`, `{{roll}}`, вложенные блоки. Парсер рекурсивного спуска на базе AST, а не регулярки.

> [!NOTE]
> Для эндпоинта Google системные инструкции приходится склеивать в одну (таков API), поэтому реальный порядок слоёв может отличаться от того, что ты выстроил в канвасе. Сверяйся с промпт-трейсером — он показывает честный финальный результат.

### Скрипты

- **Песочница JS** — `node:vm` с API, совместимым с Janitor AI. Пиши логику под конкретного персонажа, броски кубиков, отслеживание состояния.
- **ИИ-ассистент** — опиши, что нужно, простым текстом — ИИ напишет скрипт. Есть встроенные шаблоны под частые паттерны.
- **Детерминированные броски** — результаты `{{roll}}` кешируются в `context.state`. Регенерация сообщения не переворачивает кубики заново.

### Управление контекстом

- **Сводки по чату** — ручные или сгенерированные ИИ, с настройками по каждой (включать в контекст, исключать пересказанные сообщения, вкл/выкл по отдельности).
- **Авто-компакция** — «настроил и забыл»: фоновая генерация сводок каждые N сообщений.
- **Визуальный фидбек** — «Без компакции: 12,753t → Сэкономлено 11,814t (93%)».
- **Лимит сообщений в контексте** — модель видит только последние N сообщений; остаток бюджета уходит на системные слои, лор и сводки.

### Провайдеры

- **5 типов провайдеров** — OpenAI-совместимый (OpenRouter, DeepSeek, Groq, xAI, Mistral и др.), Anthropic, Google, Ollama, llama.cpp.
- **Реестр протоколов** — добавить провайдер — это один объект и одна строка. Никаких лестниц из `switch`.
- **Сэмплеры по модели** — сохраняй разные temperature/top-p/стоп-последовательности на каждую модель в одном профиле провайдера.
- **Избранные модели** — добавляй модели в избранное и переключай на лету прямо из шапки чата (работает в рамках одного провайдера — для простоты).
- **Model-aware logit bias** — fail-closed: включается только для известных пар провайдер/модель/токенайзер. Никаких тихих падений.
- **Проверка соединения** — проверь API-ключ и посмотри доступные модели до сохранения.
- **Размер ответа, контекст и reasoning** — в основных настройках задаёшь размер ответа модели и размер контекста (закрепи его, чтобы не уплывал — по умолчанию подгружается максимум выбранной модели), отображение рассуждений и их усилие (reasoning effort). Сэмплеры для продвинутых спрятаны в отдельный аккордеон.

### Персоны

- **Несколько пользовательских идентичностей** — переключай персоны в каждом чате. У каждой своё имя, описание, местоимения и аватар.
- **Описание аватара в промпте** — опционально сгенерируй через vision описание внешности аватара персонажа/персоны — и оно инъектируется в промпт отдельным слоем.
- **Быстрое переключение** — меняй персону из шапки чата, не открывая настройки.
- **Лорбуки на персону** — крепи мироустройство, которое активируется только при конкретной персоне.

### Мобильный доступ и удалённый доступ

- **Доступ по QR-коду** — открой **Mobile Access** в веб-интерфейсе, отсканируй QR → общайся с телефона в той же LAN или через Tailscale/VPN.
- **Токен-аутентификация** — удалённый API работает fail-closed: клиентам в LAN/Tailscale нужен актуальный токен, а локальный `127.0.0.1` остаётся без пароля.
- **Адаптивный UI** — это не десктоп, запихнутый в телефон. Мобильный слой построен с нуля: bottom sheets, тач-карусели, свайп-жесты.
- **Android APK** — нативная сборка под Termux. Скрипт установки берёт на себя основную настройку.

### Медиа-галерея

- **Вложения-картинки** — отправляй изображения в чат с AI-описанием через vision (авто-описание при отправке, переописание из лайтбокса).
- **Лайтбокс** — полноэкранный просмотр с зумом, панорамированием и редактированием описания.

### i18n

- **Английский + русский** — полностью переведено, на основе реестра. Добавить язык — это один JSON-файл и одна строка в реестре.

## Быстрый старт 🚀

### Windows

**Вариант 1 — Один исполняемый файл:**

Скачай последний `.exe` из [Releases](https://github.com/Noineri/vibe_tavern/releases). Запусти. Vibe Tavern откроется в браузере.

**Вариант 2 — Скрипт:**

Скачай zip-архив из [Releases](https://github.com/Noineri/vibe_tavern/releases), распакуй, запусти `Start Vibe Tavern.bat`.

### Linux

Скачай tar.gz архив из [Releases](https://github.com/Noineri/vibe_tavern/releases), распакуй, запусти `Vibe_Tavern.sh`

Или склонируй весь репозиторий и запусти вручную:

```bash
git clone https://github.com/Noineri/vibe_tavern.git
cd vibe_tavern
bun run dev
```

### macOS 

На данный момент у нас нет готовых сборок для macOS. Но можно использовать git clone метод из Linux инструкции выше

### Docker

```bash
docker compose up -d
```

### Android (Termux)

APK-сборка под Termux — автоматизирует большую часть установки. См. [Гайд по установке на Android](docs/android-setup-ru.md).

## Архитектура 🏗️

Vibe Tavern — это однопроцессный монолит: React 19 SPA + Hono API + SQLite (режим WAL), всё из одного процесса Bun. Никаких микросервисов, отдельного сервера БД и сложностей развёртывания.

```
vibe_tavern/
├── apps/web/                 # React 19 SPA (Vite)
├── services/api/             # Hono-бэкенд (Bun.serve)
├── packages/domain/          # Фундамент без зависимостей: типы, брендированные ID, константы
├── packages/api-contracts/   # Zod-схемы, общие для фронта и бэка
├── packages/db/              # Drizzle ORM (SQLite WAL) + сторы сущностей
├── packages/prompt-pipeline/ # Чистая сборка промптов + движок макросов (без I/O)
├── packages/import-export/   # Парсеры карточек/чатов/лорбуков SillyTavern V2/V3
└── data/                     # Рантайм-данные (БД, персонажи, ассеты) — gitignored
```

Строгий граф зависимостей, без циклических импортов, TypeScript strict на всём пути. Конвейер промптов — чистая функция: без I/O, полностью тестируемая. Wire-DTO-контракты живут в одном пакете, поэтому расхождение типов между фронтом и бэком — это ошибка компиляции, а не рантайм-баг.

Полная архитектурная документация — в [`docs/architecture/`](docs/architecture/).

## Roadmap 🛣️

Vibe Tavern в активной разработке. Что будет дальше:

### Ближайшее время
- **Групповые чаты** — оркестрация нескольких персонажей с изолированными промптами, скрытыми слоями и серверным управлением очерёдностью. [Дизайн готов](docs/architecture/decisions.md).
- **Novel Mode** — написание прозы с редактором TipTap, плоской text-completion, взаимодействием по абзацам и стримингом ghost text. [Дизайн готов](docs/architecture/decisions.md).

### Средняя перспектива
- **Agentic Mode** — генерация роем: N параллельных творческих черновиков → структурный критик (6-осевая рубрика) → писец по стратегии. Превращает процесс генерации в видимый, аудируемый артефакт. [Дизайн готов, Phase 0 валидирован](docs/architecture/decisions.md).
- **Система плагинов** — расширяемость для фич от комьюнити.

### Долгая перспектива
- Интеграция генерации изображений (A1111 / ComfyUI)
- TTS / STT
- Векторный/RAG-поиск
